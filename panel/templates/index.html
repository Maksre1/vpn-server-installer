Cloudflare WARP активен

Весь трафик сервера автоматически идёт через сеть Cloudflare WARP. Она работает быстро и не режет скорость.

Зачем это нужно? Многие хостинг-провайдеры выдают IP-адреса из российских подсетей. Даже если формально сервер определяется как зарубежный, крупные сайты и нейросети (вроде ChatGPT или Claude) увидят российские корни подсети и заблокируют доступ. WARP решает эту проблему — сайты будут видеть чистый международный IP-адрес.

Как это устроено?

1. Основной исходящий маршрут (Default Route)

Это путь, по которому сервер будет отправлять весь обычный трафик по умолчанию. Вы можете выбрать:

  • Напрямую (IP сервера) — без туннелирования.
  • Cloudflare WARP — через сеть Cloudflare для смены внешнего IP сервера.
  • Proton VPN — через выбранный или автоматический сервер Proton.

2. Персональные маршруты доменов

Нажмите «Настроить маршруты доменов», чтобы переопределить путь для конкретных сайтов. Например:

  • Пустить все сайты через WARP, но для instagram.com задать маршрут через Proton VPN.
  • Пустить все сайты через Proton, но российские ресурсы направить напрямую через Напрямую (IP сервера).

3. Каскадирование служб (Исходящие каналы)

Это настройка для трафика, который пользователи отправляют на ваш сервер через Hysteria, Mieru и др. Каждую службу можно завернуть в свой каскад (WARP или Proton), независимо от настроек основного маршрута.

Подсказка: После изменения любых правил маршрутизации не забудьте нажать зеленую кнопку «Применить настройки» внизу страницы, чтобы сервер обновил конфигурацию Sing-Box.

Расширенный режим

В этом режиме можно подключить Proton VPN или Amnezia, настроить цепочки обхода (каскад) и персональные маршруты для сайтов.

Настройки нужны только если вы точно понимаете, зачем они вам.

Для обычной работы стандартного режима (когда всё идёт через WARP) хватает с головой — это самый быстрый и стабильный вариант.
